# SE_DAY4_Software-Project-Management
## 1. Why is timely delivery crucial in software project management, and how can project managers ensure that deadlines are met?

Timely delivery is super important because delays can increase costs, frustrate clients, and even make the software outdated before it's launched.
Project managers can ensure deadlines are met by doing the following: Set realistic deadlines, Use project management tools like Trello  to track progress, Break work into smaller tasks to make things manageable ,Communicate regularly with the team to catch any roadblocks early and Prioritize tasks so the most important features get done first.

## 2. How does effective cost control contribute to the success of a software project? What strategies can be used to prevent budget overruns?

Effective cost control contributes to the success of a software project because if a project goes over budget, companies might run out of money, cut important features, or even cancel the project. 
To prevent budget overruns  teams can: Plan a clear budget upfront and stick to it, Track expenses in real time so there are no surprises ,Use Agile methods to adapt and avoid wasting time and money on unnecessary features
Have a buffer for unexpected costs.

## 3. Compare and contrast Agile and Waterfall methodologies. What are the main advantages and disadvantages of each?

Agile is flexible and great for projects where things might change. It allows teams to adjust as they go, but it can be chaotic if not managed well.
Waterfall is more structured, with a step by step approach. It works best  when everything is well planned from the start, but changes are hard to make later on.
Advantages of Agile	are; Flexible, can change as needed, Works in small sprints and is Good for evolving projects	
Disadvantages of Agile are; Can be messy if not managed well.
Advantages of Waterfall are; Fixed plan, hard to change, Follows a step by step process and it is ood for projects with clear requirements.
Disadvantages are; Can be slow and rigid

## 4. In what types of projects might Agile be more beneficial than Waterfall, and vice versa? Can you provide examples of each?

Agile works best for software startups, mobile apps, or anything where feedback and quick changes matter. Example: A new social media app where user feedback can shape features.
Waterfall is better for projects with clear requirements, like government systems or large infrastructure software. 

## 5. What are some methods for ensuring quality assurance throughout a software project? Why is it important to maintain high standards?

Methods for ensuring quality assurance throughout a software project include;
Testing early and often e.g unit tests, integration tests.
Automated testing to catch bugs faster.
Code reviews so developers double check each other’s work.
Beta testing with real users before launch.
Following coding standards to keep things clean and readable.
It is important to maintain high quality assurance standards because when the software contains bugs users become unhappy with the software and stop using it.

## 6. How does defining the project scope contribute to successful project planning? What is a Work Breakdown Structure (WBS), and why is it useful?

Defining the project scope contributes to successful project planning by  preventing  scope creep that is when a project keeps expanding and never finishes. This helps  keep everyone on the same page about what’s included.

A Work Breakdown Structure is a breakdown of the project into smaller, manageable tasks. 
A work breakdown structure is useful because;It makes planning easier, Helps track progress and It ensures no tasks are forgotten
